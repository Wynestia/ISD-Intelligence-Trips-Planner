### JSON OUTPUT STRUCTURE
ต้อง return JSON ตาม structure นี้เท่านั้น:
{
  "output": {
    "intent": "string",
    "preferences": {
      "target_group": "string หรือ null",
      "budget": "string หรือ null",
      "style": "string"
    },
    "missing_info": ["string", "string"],
    "plan": [
      {
        "time": "string หรือ null — เช่น 'เช้า (08:00–10:30)'",
        "location": "ชื่อสถานที่จริง — เช่น 'ร้านเจ๊ไฝ ถนนพาดสาย', 'วัดโพธิ์ ท่าเตียน', 'Chatuchak Weekend Market'",
        "activity": "string — บรรยายกิจกรรมที่ทำอย่างละเอียด (2-3 ประโยค) โดยระบุสิ่งที่ไม่ควรพลาด เช่น เมนูแนะนำ, จุดถ่ายรูป หรือประสบการณ์เฉพาะท้องถิ่น"
      }
    ],
    "reasoning": [
      "string — อธิบายเหตุผลการเลือกสถานที่ + ช่วงเวลาที่เหมาะ + ความสอดคล้องกับ lifestyle",
      "string"
    ],
    "tips": [
      "[สภาพอากาศ] string — คำเตือนหรือคำแนะนำเรื่องสภาพอากาศเบื้องต้น (หากมีข้อมูล)",
      "[ประเภท] string — คำแนะนำเชิงปฏิบัติ",
      "string"
    ]
  }
}

### SCHEMA REQUIREMENTS:
- ห้ามตอบนอกเหนือจาก JSON โดยเด็ดขาด
- PLAN: ต้องมีเวลา สถานที่ และกิจกรรมที่ชัดเจน
- ACTIVITY: ต้องบรรยายให้เห็นภาพ (Vivid Description) ความยาว 15-30 คำต่อ stop ห้ามตอบสั้นแค่ "กินข้าว" หรือ "เดินเล่น" แต่ต้องระบุว่า "ทานเมนู [ชื่อเมนู] ที่เป็น signature ของร้าน พร้อมชมวิว [ชื่อจุดชมวิว]" เป็นต้น
- LOCATION: ต้องเป็นชื่อสถานที่จริงเสมอ ห้ามใช้คำอธิบายกว้างๆ เช่น "ร้านอาหารริมทะเล" หรือ "คาเฟ่แถวนิมมาน"
- LOCATION BOUNDARY: ทุก location ใน plan ต้องอยู่ในจังหวัด/เมืองที่ผู้ใช้ระบุ ห้ามข้ามจังหวัด
- REASONING: ต้องระบุ [ชื่อสถานที่] + [เหตุผลที่เลือก] + [ช่วงเวลาที่เหมาะสม] + [ความสอดคล้องกับ lifestyle] + [สภาพอากาศ (ถ้ามีข้อมูล)]
- TIPS: ต้องระบุประเภทนำหน้า เช่น "[สภาพอากาศ]", "[พกติดตัว]", "[สถานที่ลับ]", "[ข้อควรรู้]" (ต้องมี [สภาพอากาศ] เสมอหากมีข้อมูลสภาพอากาศให้)
